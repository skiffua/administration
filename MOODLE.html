Резервне копіювання Moodle

Інструкція з повного бекапу системи Moodle 4.4.3 (moodle-ua.free.nf) через панель керування хостингом та FTP без використання командного рядка.

1. Специфікація даних для архівації

Для повного відновлення платформи у разі збою або міграції критично важливо мати три складові компоненти:

Компонент Статус для бекапу Метод збереження
База даних (SQL) Обов'язково Експорт через phpMyAdmin в один клік
Папка /filedir Обов'язково Всі завантажені файли, тести, картинки (копіювати повністю)
Папка /lang Бажано Локалізовані мовні пакети (важить мало, можна зкопіювати швидко)
Кеш (/cache, /localcache) Ігнорувати Тимчасові файли. Не включати в архів (уповільнюють процес)

2. Покроковий алгоритм дій

  1. Експорт Бази Даних:
    Зайти в панель хостингу, відкрити інструмент phpMyAdmin. Вибрати базу даних сайту у лівій колонці. Перейти на вкладку "Експорт" (Export). Обрати спосіб "Швидкий", формат "SQL" та натиснути кнопку "Вперед" (Go).
  2. Збереження файлового сховища (mdata):
    Через "Файловий менеджер" хостингу або FTP-клієнт (FileZilla) перейти у директорію moodledata (або mdata).
    • Виділити та заархівувати папку /filedir. Якщо сервер безкоштовного хостингу зависає через великий об'єм файлів, скачувати вміст напряму через FTP частинами.
    • Аналогічно заархівувати або скачати папку /lang.
  3. Бекап вихідного коду сайту:
    У папці веб-сервера (наприклад, public_html або moodle) створити `.zip` архів усього вмісту. Обов'язково перевірити наявність файлу config.php у корені архіву.

Локальна директорія для збереження готових бекапів у комп'ютерному класі: \\Komp0\archiv\backup\Moodle_[ДАТА]

3. Швидка перевірка поточної версії системи

  • Через інтерфейс (Стан системи / Плагіни):
    Блок Адміністрування сайту → розділ "Стан системи" (System status) або "Плагіни" (Plugins) → "Огляд плагінів". Точна версія Moodle відображається у верхній частині робочої області.
  • Альтернативний метод (у футері):
    Прокрутити будь-яку сторінку сайту до самого кінця. У стандартних темах версія збірки дублюється дрібним шрифтом у футері (нижній панелі) поруч із документацією.
  • Через серверні файли:
    Переглянути вміст файлу version.php в кореневій директорії сайту. Точний реліз прописаний у рядку зі змінною $release.
