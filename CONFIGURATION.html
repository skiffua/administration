Конфігурація сервісів сервера

Технічний опис налаштувань, шляхів конфігурації та параметрів розгортання локальних сервісів.

Глобальні дані сервера

IP-адреса: 192.168.14.240
Логін: skynet
Пароль: s1111

1. Cockpit (Веб-панель)

Використовується для веб-доступу до керування сервером (термінал, логи, моніторинг).

Адреса сервісу: https://192.168.14.240:9090/

Для авторизації використовувати глобальний логін та пароль сервера (skynet / s1111).

2. SQUID Proxy (Фільтрація трафіку)

Шлях на сервері: /etc/squid

Файл конфігурації: /etc/squid/squid.conf

Вміст конфігураційного файлу squid.conf

# Вказуємо порт (обов'язковий для проксі)
http_port 3128

visible_hostname localhost

# Ваше правило блокування ігрових доменів
acl gaming_domains dstdomain "/etc/squid/gaming_sites.txt"

# Вказуємо шлях до нашої кастомної папки помилок
error_directory /usr/share/squid/errors/custom

# Вказуємо сторінку помилки для цього ACL (локальний файл або редірект)
# deny_info block_page.html gaming_domains
deny_info http://192.168.14.240:8081 gaming_domains

# Сама заборона доступу
http_access deny gaming_domains

Скрипти керування проксі-сервером (Debian)

Для перезапуску сервісу після редагування списків або повної зупинки фільтрації безпосередньо на сервері:

  • Перезапуск (застосування змін): sudo systemctl restart squid або перечит конфігу без розриву з'єднань: sudo squid -k reconfigure
  • Зупинка сервісу: sudo systemctl stop squid
  • Запуск сервісу: sudo systemctl start squid

3. База даних MySQL (MariaDB + Adminer)

Веб-доступ (Adminer): http://192.168.14.240:8082/

Шлях до проекту на сервері: /db

Файл конфігурації docker-compose.yml

services:
  db:
    # Використовуємо версію 10.5, вона дуже невибаглива
    image: mariadb:10.5
    container_name: light_db
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: root
      MYSQL_DATABASE: test_db
    ports:
      - "3306:3306"

  adminer:
    # Фіксована версія для стабільності веб-інтерфейсу
    image: adminer:4.8.1
    container_name: light_admin
    restart: always
    ports:
      - "8082:8080"
    depends_on:
      - db

4. Локальний месенджер Rocket Chat

Веб-доступ: http://192.168.14.240:3000/

Шлях до проекту на сервері: /rocketchat

Файл конфігурації docker-compose.yml

services:
  mongo:
    image: mongo:4.0
    container_name: rocketchat-mongo-1
    restart: unless-stopped
    volumes:
      - ./data/db:/data/db
    command: mongod --oplogSize 128 --replSet rs0

  rocketchat:
    image: registry.rocket.chat/rocketchat/rocket.chat:3.18.5
    container_name: rocketchat-rocketchat-1
    restart: unless-stopped
    environment:
      - PORT=3000
      - ROOT_URL=http://192.168.14.240:3000
      - MONGO_URL=mongodb://mongo:27017/rocketchat
      - MONGO_OPLOG_URL=mongodb://mongo:27017/local
    depends_on:
      - mongo
    ports:
      - 3000:3000
